Events

Events are generated automatically according to your actions, thanks to them, you can have an overview of the activity of your applications.

Event attributes

field description
id unique ID of event
created_at date of creation
user embedded user who generated the event
type type of event (can be among delete_variable deployment edit_variable new_variable restart run scale)
app_id unique ID of app the event belongs to

According to the type field, extra data will be included in the structure:

Example object:

  • Base event
{
  "id": "54dcdd4a73636100011a0000",
  "created_at": "2015-02-12T18:05:14.226+01:00",
  "user": {
      "username": "johndoe",
      "email": "john@doe.com",
      "id": "51e6bc626edfe40bbb000001"
  },
  "app_id": "5343eccd646173000a140000"
}
  • Restart event

When: The application or some containers have been restarted type=restart

field description
scope The scope of the restart, null is all

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at" : "2015-02-12T18:05:14.226+01:00",
  "user" : {
      "username" : "johndoe",
      "email" : "john@doe.com",
      "id" : "51e6bc626edfe40bbb000001"
  },
  "app_id" : "5343eccd646173000a140000",
  "scope" : ["web", "worker"]
}
  • Scale event

When: The application has been scaled out type=scale

field description
previous_containers The formation before the operation
containers The formation after the request

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at" : "2015-02-12T18:05:14.226+01:00",
  "user" : {
      "username" : "johndoe",
      "email" : "john@doe.com",
      "id" : "51e6bc626edfe40bbb000001"
  },
  "app_id" : "5343eccd646173000a140000",
  "previous_containers" : {
    "web" : 1,
    "worker": 0
  },
  "containers" : {
    "web" : 2,
    "worker" : 1 
  }
}
  • Deployment event

When: A deployment has been done type=deployment

field description
deployment The Deployment associated to the event

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at" : "2015-02-12T18:05:14.226+01:00",
  "user" : {
      "username" : "johndoe",
      "email" : "john@doe.com",
      "id" : "51e6bc626edfe40bbb000001"
  },
  "app_id" : "5343eccd646173000a140000",
  "deployment" : {
    "id" : "5343eccd646aa3012a140230",
    "push": "johndoe",
    "git_ref" : "0123456789abcdef"
  }
}
  • Run event

When: Someone runs scalingo run from the CLI type=run

field description
command The command run by the user

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at": "2015-02-12T18:05:14.226+01:00",
  "user": {
      "username": "johndoe",
      "email": "john@doe.com",
      "id": "51e6bc626edfe40bbb000001"
  },
  "app_id": "5343eccd646173000a140000",
  "command": "bundle exec rake db:migrate"
}
  • New Variable event

When: Each time a variable is added to the application type=new_variable

field description
name Name of the newly created variable
value Value of the new variable

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at": "2015-02-12T18:05:14.226+01:00",
  "user": {
      "username": "johndoe",
      "email": "john@doe.com",
      "id": "51e6bc626edfe40bbb000001"
  },
  "app_id": "5343eccd646173000a140000",
  "name" : "VAR1",
  "value" : "VAL1"
}
  • Edit Variable event

When: Each time a variable is modified type=edit_variable

field description
name Name of the modified variable
value New value of the modified variable
old_value Previous value of the modified variable

Example object:

{
  "id": "54dcdd4a73636100011a0000",
  "created_at": "2015-02-12T18:05:14.226+01:00",
  "user": {
      "username": "johndoe",
      "email": "john@doe.com",
      "id": "51e6bc626edfe40bbb000001"
  },
  "app_id": "5343eccd646173000a140000",
  "name" : "VAR1",
  "old_value" : "VAL1",
  "value" : "VAL2"
}

List the events of an app

With this list of events, you can reconstruct the timeline of an application.

GET https://api.scalingo.com/v1/apps/[:app]/events

Feature: pagination

Example

curl -H "Accept: application/json" -H "Content-Type: application/json" -u :$AUTH_TOKEN https://api.scalingo.com/v1/apps/[:app]/events

Returns 200 OK
